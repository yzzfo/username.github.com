      在购买前使用试用版

      视频捕捉是一款插件，让身为 Unity 开发者的你从 Unity 应用程序中捕捉视频和音频。非常适合为基于 Unity 的游戏或应用录制视频预告片、演示和应用内素材。而且它还快捷、灵活、易于使用。视频被录制之后，你来决定如何将它处理。给用户完整的分享权限，或仅允许应用程序中播放，又或两者之间的任何权限。

      功能：
      - 非常容易使用
      - 从任何摄像头捕捉视频
      - 用光标从屏幕捕捉视频
      - 从渲染纹理中捕捉视频
      - 在场景内捕捉音频
      - 从麦克风中捕捉音频
      - 多个编码支持（H.264、H.265、FFV1、VP8、VP9）
      - 多个摄像头同时捕捉
      - 实时直播到远程流媒体服务（Youtube、Facebook 等等）
      - 高达 8K 视频捕捉
      - 高达 16K 截图或序列图像捕捉
      - 立体声视频捕捉（左右、上下）
      - 360 视频捕捉（等距状、立面贴图）
      - 360 立体声视频捕捉（左右、上下）
      - 支持实时和离线捕捉
      - 支持线性和 Gamma 颜色空间

      支持的平台：
      - Windows（32 位或 64 位）
      - macOS（64 位）

      常见问题：
      问：视频捕捉插件需要哪个 Unity 版本？
      答：我们推荐 Unity 5.6 或更高版本。

      问：我能使用多个摄像头同时捕捉吗？
      答：是的，但你所用的摄像头越多，设备花费的资源就越大。此外，360 视频录制目前尚未支持多摄像头录制。

      问：什么是离线渲染设置？
      答：假如你启用了离线渲染设置，则捕捉会话只会一直移到下一个帧，直到当前帧记录成功。它在捕捉 8K 视频或消耗很大的视频捕捉时，对你创建流畅的视频有用。

      问：我的设备支持高性能 GPU 编码吗？
      答：GPU 编码仅在 Windows 使用，而且需要一张独立显卡。
      • 带有 Kepler、Maxwell 和 Pascall 生成 GPU 的 NVIDIA Quadro、Tesla、GRID 或 GeForce 产品。NVidia Windows 显示驱动程序：375.95 或更高版本。
      • AMD GPU 支持以下驱动程序： AMD Radeon Software Crimson 17.1.1 或更高版本。
      • Windows 7 SP1 或更高版本。

      问：录制视频的分辨率极限是多大？
      答：支持的视频分辨率会根据编码类型而不同：
      • AMD GPU 编码：最高 4K（4096 x 2048）
      • NVidia GPU 编码：最高 4K（4096 x 4096）
      • 软件编码：最高 8K（7680 x 4320）

      问：为什么我无法在 MacOS 上生成视频？
      答：如果你无法在 MacOS 上生成视频，这可能是由于程序没有允许运行 FFmpeg 的权限。如果想要修复，请在 VideoCapture 编辑器菜单中点击授予 MacOS Build 权限项。

      问：我正在使用 MacOS 和 Unity 2017，为什么我无法导入资源？
      答：Video Capture Pro 资源包含样本 wav 音频文件。 MacOS 版本 Unity 2017 有一个导入 wav 文件的错误，除去这些 wav 音频文件你就可以成功导入资源。

      问：如何用视频捕捉来记录游戏 UI ？
      答：请将捕捉源更改到为 SCREEN，你也可以试用演示 “Demo_02_FromScreen” 来记录游戏 UI。

      问：我正在使用 Windows，为什么在启用 GPU 编码后程序自动退出？
      答：在用 GPU 捕捉时可能会出现意外，请确保你正在使用的是 Direct3D11 Windows Graphic API 。
